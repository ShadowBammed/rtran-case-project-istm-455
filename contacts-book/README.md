# Contacts Book

A minimal REST API for managing contacts with full CRUD operations, backed by SQLite, served over HTTPS with a self-signed certificate, and containerised with Docker.

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | Node.js 22 |
| Framework | Express 5 |
| Database | SQLite via `better-sqlite3` |
| Package manager | npm |
| Testing | Vitest + Supertest |
| Containerisation | Docker + Docker Compose |
| TLS | Self-signed certificate (OpenSSL) |

## Project Structure

```
contacts-book/
├── src/
│   ├── app.js          # Express app factory
│   ├── server.js       # HTTPS entry point
│   ├── db.js           # SQLite setup + seed data
│   └── routes/
│       └── contacts.js # CRUD route handlers
├── test/
│   └── contacts.test.js
├── certs/              # Generated by npm run gen-cert (git-ignored)
├── Dockerfile
├── docker-compose.yml
├── gen-cert.sh
└── package.json
```

## Prerequisites

- Node.js 22+
- npm 10+
- OpenSSL (for TLS certificate generation)
- Docker & Docker Compose (for containerised deployment)

## Install

```bash
npm install
```

## Generate TLS Certificate

Creates `certs/server.pem` — a single PEM file combining the private key and self-signed certificate.

```bash
npm run gen-cert
```

The script calls OpenSSL to produce a 4096-bit RSA key pair valid for 365 days, then concatenates `key.pem` and `cert.pem` into a single `server.pem`.

## Run

### HTTPS server (production mode)
```bash
npm start          # listens on https://localhost:3443
```

### HTTP server (dev mode, no cert needed)
```bash
npm run dev        # listens on http://localhost:3000
```

## Test

```bash
npm test           # run all tests once
npm run test:watch # watch mode
```

## API Endpoints

| Method | Path | Description |
|---|---|---|
| GET | /health | Health check |
| GET | /contacts | List all contacts |
| GET | /contacts/:id | Get one contact |
| POST | /contacts | Create a contact |
| PUT | /contacts/:id | Replace a contact |
| DELETE | /contacts/:id | Delete a contact |

### Example: Create a contact
```bash
curl -sk -X POST https://localhost:3443/contacts \
  -H 'Content-Type: application/json' \
  -d '{"name":"Jane Doe","email":"jane@example.com","phone":"555-1234"}'
```

### Example: List all contacts
```bash
curl -sk https://localhost:3443/contacts | jq .
```

## Docker

### Build and run with Docker Compose
```bash
npm run docker:up
```

### Stop
```bash
npm run docker:down
```

The compose file mounts a named volume (`contacts-data`) so the SQLite database persists across container restarts.
